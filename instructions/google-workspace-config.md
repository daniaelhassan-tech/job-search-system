## Tool-Specific Protocols

### Meeting Transcript (Granola)

**Post-call debrief:** After detecting a call ended, check Granola for the meeting transcript. Use it to:
- Extract verbatim quotes and specific claims (don't paraphrase when precision matters)
- Identify commitments made by either side
- Catch details I might not mention in my verbal debrief

### Email (Gmail)

**Periodic sync:** Search for recent messages from contacts in the pipeline (names are in state.md and dossiers.md). Look for:
- Scheduling confirmations
- Recruiter updates
- Follow-ups I haven't responded to
- New inbound outreach from people/companies not yet in the system

**Draft tracking:** Check for unsent drafts related to pipeline companies. Drafts that sit >3 days are aging — flag them.

### Calendar (Google Calendar)

**Upcoming prep:** Pull events for the next 5 days. Flag any that match pipeline companies or contacts. For each:
- Check whether the dossier has been updated recently
- Check whether there are open questions from the last interaction that need prep
- Flag if no prep has been done

**Post-call detection:** Check for meetings that ended in the last 1-2 hours. Cross-reference with pipeline companies to trigger debrief protocol.

## Alternative Tool Mapping

If you don't use these specific tools, here's what to swap:

| This config uses | Purpose | Alternatives |
|---|---|---|
| Gmail | Email scanning, draft tracking | Outlook, Superhuman, any email client with API/MCP access |
| Google Calendar | Meeting detection, upcoming prep | Outlook Calendar, Cal.com, Fantastical |
| Granola | Meeting transcripts | Zoom, Teams, Otter.ai, Fireflies, Fathom, manual notes |

The protocols above work with any tool that serves the same purpose — just update the tool references in your instructions.
